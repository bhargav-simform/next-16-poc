# Code Analyzer

Paste a public GitHub repository URL and get static analysis, local AI-generated insights, and a
retrieval-augmented chat over the codebase — all running locally, with no paid APIs.

## Features

1. **Repository ingestion** — validates a GitHub URL, shallow-clones the repo into a local
   workspace, and extracts basic metadata (files, languages, latest commit, `package.json`).
2. **Static code analysis** — four independent analyzers run over the cloned repo:
   - *Dependency* — detected framework, libraries, scripts
   - *Structure* — architecture pattern (App Router / Pages Router / hybrid), folder layout
   - *Code quality* — large files, complex components, duplicate imports, console/TODO comments,
     unused files
   - *Security* — committed secrets, `.env` files, risky patterns (`eval`, `dangerouslySetInnerHTML`,
     leaked non-public env vars in client components)

   Results are merged into one weighted score (0–100) and persisted per repository.
3. **AI insights (local, via Ollama)** — streamed narrative summary, prioritized recommendations,
   and per-issue explanations generated by `qwen2.5-coder`.
4. **Repository Q&A (RAG)** — the repo is chunked, embedded with `nomic-embed-text`, and stored in
   Postgres with `pgvector`. Questions are answered by retrieving the most relevant chunks and
   passing them to `qwen2.5-coder` as grounding context.
5. **Refresh analysis** — re-runs static analysis against the already-cloned workspace and
   invalidates the cached report, without re-cloning or re-indexing.

## Tech stack

| Layer | Technology |
|---|---|
| Framework | [Next.js 16](https://nextjs.org) (App Router, Turbopack, React Compiler) |
| UI | React 19, Tailwind CSS v4, [shadcn/ui](https://ui.shadcn.com) (`base-ui` primitives), `lucide-react` |
| Language | TypeScript (strict) |
| Database | PostgreSQL 16 + [pgvector](https://github.com/pgvector/pgvector) (via `pg`, raw SQL — no ORM) |
| Local AI | [Ollama](https://ollama.com) — `qwen2.5-coder:7b` (chat/completion), `nomic-embed-text` (embeddings) |
| Git operations | `simple-git` |
| Package manager | pnpm |

### Notable Next.js 16 features in use

See [`NEXTJS_FEATURES.md`](./NEXTJS_FEATURES.md) for the full breakdown with file references.

- **Server Components** — the repository dashboard (`app/repo/[id]/page.tsx`) reads persisted data
  directly at render time, no client fetch.
- **Server Actions** — repository import and "Refresh analysis" are both Server Actions.
- **Streaming UI** — AI responses (summary, recommendations, explain, chat) stream token-by-token
  from Ollama through Route Handlers.
- **Suspense** — `app/repo/[id]/loading.tsx` provides an instant loading skeleton while the report
  is fetched/computed.
- **Cache Components (`"use cache"`)** — the repository report read is cached with `cacheLife` and
  `cacheTag`.
- **Revalidation** — `updateTag` gives read-your-own-writes semantics when refreshing analysis.

## Prerequisites

- Node.js 22+ and [pnpm](https://pnpm.io)
- [Docker](https://www.docker.com) (for Postgres + pgvector)
- [Ollama](https://ollama.com) installed and running locally

## Getting started

```bash
# 1. Install dependencies
pnpm install

# 2. Start Postgres (pgvector) via Docker Compose
docker compose up -d

# 3. Copy the env template and adjust if needed
cp .env.example .env

# 4. Pull the required Ollama models
pnpm ollama:pull        # qwen2.5-coder:7b — chat / analysis / recommendations
pnpm ollama:pull-embed  # nomic-embed-text — embeddings for repository Q&A

# 5. Run the dev server
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000), paste a public GitHub repository URL, and
ingest it.

### Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `DATABASE_URL` | `postgres://code_analyzer:code_analyzer@localhost:5433/code_analyzer` | Postgres connection string |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama server URL |
| `OLLAMA_MODEL` | `qwen2.5-coder:7b` | Chat/completion model |
| `OLLAMA_EMBEDDING_MODEL` | `nomic-embed-text` | Embedding model for Q&A indexing |

### Useful scripts

| Script | Description |
|---|---|
| `pnpm dev` | Start the Next.js dev server (Turbopack) |
| `pnpm build` / `pnpm start` | Production build / start |
| `pnpm lint` | Run ESLint |
| `pnpm ollama:pull` | Pull the `qwen2.5-coder:7b` model |
| `pnpm ollama:pull-embed` | Pull the `nomic-embed-text` embedding model |
| `pnpm ollama:start` | Run `ollama serve` in the foreground |
| `pnpm ollama:stop` | Stop the Ollama systemd service |

## Project structure

```
actions/            Server Actions (ingest, reanalyze)
app/
  page.tsx          Ingestion landing page
  repo/[id]/         Repository dashboard (Server Component) + Suspense loading state
  api/ai/*/route.ts  Streaming Route Handlers (analyze, explain, recommend, qa)
components/         UI components (shadcn/ui primitives + feature components)
services/           Business logic: git clone, analyzers, Ollama client, indexing, DB access
lib/                Shared constants and utilities
types/              Shared TypeScript types
```
